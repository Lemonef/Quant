RTN-Core — Rotational Trend (candidate)

Rotational top-8 crypto trend · 4H · BTC/MA200 risk-on gate · equal-weight · funding-aware sizing · point-in-time universe. STATUS: candidate — backtest pending, NOT live

Thesis. Hold the strongest-trending liquid coins from a survivorship-bias-free universe, only when the market regime is risk-on, sized down when funding is hot. It upgrades the validated Donchian/MA200 edge through better construction (universe + selection + sizing) — not new indicators. Origin: 2026-06-10 multi-agent review (Claude Opus 4.8 + Codex GPT-5.5 + Gemini 2.5 Pro).

Construction

Universe (PIT)Survivorship-bias-free: listing-date gating (+30d), rolling 30d median $ADV ≥ ~$20M, delisting exit at last price, symbol-migration mapping, perp/funding availability for the leveraged version. Eligibility rebuilt weekly.
SignalRank eligible coins by trend strength (Donchian 55/20 breakout + normalized momentum). Hold top 8, equal-weight. Evaluate on the 4H close. Rank-hysteresis buffer (swap only if a challenger beats a held coin by ≥2 ranks) to cut churn.
Regime gateBTC > MA200 = risk-on (the validated master filter that halved 2022 losses). Risk-off → cash, no new entries.
SizingATR risk sizing (~0.5–1%/trade); funding-aware throttle (cut size when perp funding hot — cost/risk, not alpha); leverage ≤1.5x core / ≤2x high-CAGR; per-coin ≤15%, capped if top-N highly correlated.
Risk / roleDonchian-20 / ATR stop; max-DD guardrail de-grosses on breach. v1 = core role 1–1.5x. Keep the existing Donchian core running as the benchmark.

Validation plan (gating — before any capital)

  • 1. Point-in-time universe rebuild + OOS re-run FIRST — both external models flagged the basket may be a data-construction artifact until rebuilt PIT.
  • 2. Funding-aware sizing — cheap, high-certainty risk fix for the leveraged/futures version.
  • 3. Rotational top-N — the alpha bet, only AFTER the universe is validated. Compare vs the existing Donchian core (not just BTC): must win on Sharpe + maxDD net of fees, not raw CAGR.
  • 4. Paper-first hardening — 1–3 months paper, track fees/slippage/funding/stale-data/failed-runs, add kill-switches, THEN deploy as a separate live bot.

Known risks

Rotational top-N can collapse into overfit cross-sectional momentum without strict walk-forward + regime gating · N / ranking blend / ADV threshold are tunable knobs = overfit surface (use round numbers, test sensitivity) · 4H closes can't see liquidation-wick deleveraging (keep leverage low) · the universe is the biggest single risk — if it isn't truly PIT, everything downstream inherits the artifact.
